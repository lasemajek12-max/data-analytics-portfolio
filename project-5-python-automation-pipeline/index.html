5 Layers of Quality Checking

🗂️
Schema Validation
Missing or misspelled column names caught before any processing begins. Fails fast with a clear error message.
🔢
Type Enforcement
Dates stored as strings, prices stored as text, IDs stored as floats — all corrected automatically.
🕳️
Null Handling
Critical fields (price, date, ID) drop the row. Non-critical fields (category, region) get imputed. Both are logged.
👥
Duplicate Removal
Exact row duplicates and duplicate transaction IDs are two separate checks — one catches full copies, one catches re-submitted records.
⚖️
Business Rules
Negative unit prices, discount_pct over 100, quantity of zero, and future transaction dates are all flagged to a separate output file with a reason column.
📋
Full Audit Log
Every step is logged with a timestamp, level (INFO / WARNING), and count — giving a complete record of what was changed and why.

Sample — Business Rule Validation

🐍 retail_data_pipeline.py · Step 6
def validate_business_rules(df):
    """Flag rows violating business logic — saved separately with reason."""
    flagged_rows = []

    # Rule 1: Negative unit prices are impossible
    neg_price = df[df['unit_price'] < 0].copy()
    neg_price['flag_reason'] = 'Negative unit price'
    flagged_rows.append(neg_price)

    # Rule 2: Discount over 100% makes no business sense
    bad_disc = df[df['discount_pct'] > 100].copy()
    bad_disc['flag_reason'] = 'Discount exceeds 100%'
    flagged_rows.append(bad_disc)

    # Rule 3: Zero quantity — no transaction actually happened
    zero_qty = df[df['quantity'] <= 0].copy()
    zero_qty['flag_reason'] = 'Zero or negative quantity'
    flagged_rows.append(zero_qty)

    # Rule 4: Future-dated records — data entry error
    future = df[df['txn_date'] > pd.Timestamp('today')].copy()
    future['flag_reason'] = 'Future transaction date'
    flagged_rows.append(future)

    flagged = pd.concat(flagged_rows).drop_duplicates()
    clean   = df[~df.index.isin(flagged.index)]

    logging.warning(f"  {len(flagged)} rows flagged with business rule violations")
    return clean, flagged
